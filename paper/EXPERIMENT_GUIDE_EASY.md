# COP-RFS 논문 실험 가이드 — 쉬운 설명
### (각 실험의 파라미터 의미와 이론을 초등학생도 이해할 수 있도록)

---

## 기본 개념: "소리가 어디서 오는지 찾기"

우리 귀가 2개라서 소리 방향을 알 수 있듯이, **안테나(센서)를 여러 개** 놓으면 전파가 어디서 오는지 알 수 있습니다. 이것을 **DOA (Direction of Arrival) 추정**이라고 합니다.

### 핵심 비유

| 논문 용어 | 초등학생 비유 |
|-----------|-------------|
| **센서 (M=8)** | 교실에 놓인 8개의 귀 (마이크) |
| **신호원 (K)** | 동시에 말하는 아이들 수 |
| **DOA (θ)** | 말하는 아이가 어느 방향에 있는지 (각도) |
| **SNR (dB)** | 교실이 얼마나 조용한지 (높을수록 조용 → 소리 잘 들림) |
| **스냅샷 (T)** | 녹음 시간 (길수록 더 정확) |
| **COP (Higher-Order Cumulant)** | 특별한 청력 보조기. 보통 귀로는 7명까지만 구분하지만, COP를 쓰면 14명까지 구분 가능 |
| **PHD 필터 (트래커)** | 움직이는 아이들을 눈으로 계속 따라가는 것 |
| **CRLB** | "아무리 잘해도 이 정도 오차는 생겨" 라는 이론적 한계 |

## 공통 파라미터 (모든 실험에 등장)

### `M = 8` (센서 수)
- **의미**: 안테나(귀)가 8개
- **쉬운 설명**: 교실에 마이크를 8개 놓았다고 생각하세요
- **왜 8개?**: 보통 방법(MUSIC)은 M-1 = 7개의 소리까지만 구분 가능. 우리 방법(COP)은 14개까지!
- **이론**: 안테나가 많을수록 방향을 정확히 알 수 있지만, 비용이 올라감

### `d = 0.5λ` (센서 간격)
- **의미**: 안테나 사이 거리 = 전파 파장의 절반
- **쉬운 설명**: 줄자로 잴 때, 물결 하나 크기의 반만큼 떨어뜨려 놓음
- **왜 절반?**: 물리학적으로 이 간격이 가장 깨끗하게 방향을 잡아냄 (Nyquist 조건)

### `ρ = 2` (큐뮬런트 차수)
- **의미**: 4차 통계량 사용 (2ρ = 4)
- **쉬운 설명**: 보통은 "평균"과 "분산"만 보지만, 우리는 "뾰족한 정도"와 "비대칭 정도"까지 봄
  - ρ=1 → 보통 방법 (2차 통계 = 분산) → 최대 7개 구분
  - ρ=2 → COP 방법 (4차 통계) → 최대 14개 구분
  - ρ=3 → 더 강한 방법 (6차 통계) → 최대 21개 구분 (하지만 더 많은 데이터 필요)
- **이론**: 차수가 높을수록 "가상 안테나"가 늘어남. M_v = ρ(M-1)+1 = 15개

### `scan_angles` (탐색 각도)
- **의미**: -90°에서 +90°까지 0.1° 간격으로 모든 방향을 살펴봄
- **쉬운 설명**: 머리를 왼쪽 끝에서 오른쪽 끝까지 천천히 돌리면서 "여기서 소리 나나?" 확인

### `n_trials` (몬테카를로 시행 수)
- **의미**: 같은 실험을 여러 번 반복
- **쉬운 설명**: 주사위를 1번만 던지면 결과를 못 믿지만, 100번 던지면 평균이 정확해짐
- **논문에서**: 5~10회 사용 (계산 시간 고려)

### 실험 0: 공간 스펙트럼 비교
**"누가 더 잘 듣는지 비교"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| K = 10 | 10명이 동시에 말함 | 센서(8개)보다 말하는 사람(10명)이 더 많음! |
| M = 8 | 마이크 8개 | 보통 방법으로는 7명까지만 구분 가능 |
| SNR = 15 dB | 꽤 조용한 교실 | 소리가 잡음보다 약 30배 강함 |
| T = 256 | 녹음 256회 | 중간 정도 녹음 시간 |

**결과 해석**:
- **MUSIC**: 7명까지만 들을 수 있어서, 10명 중 3명은 아예 못 찾음 → 엉뚱한 방향에 가짜 봉우리 생김
- **Capon**: 마찬가지로 실패, 봉우리가 뭉개져서 구분 불가
- **COP-4th**: 14명까지 들을 수 있으므로, 10명 전부 정확히 찾아냄! → 뾰족한 봉우리 10개

**이론**: COP가 잘 되는 이유는 "가상 안테나 확장". 8개 안테나인데 마치 15개인 것처럼 계산. 이것은 4차 큐뮬런트가 안테나 쌍의 조합을 만들어내기 때문.

### 실험 0b: COP 패밀리 스펙트럼
**"기본 COP, T-COP, SD-COP 세 형제 비교"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| K = 16 | 16명 동시 발화 | COP 한계(14명)도 넘었음! |

**세 형제**:
- **COP (기본)**: 한 번만 듣고 판단 → 16명 중 4명만 찾음
- **T-COP (시간 누적)**: 10번 연속 듣고 합산 → 12명 찾음. 여러 번 들으니 더 정확!
- **SD-COP (순차 제거)**: 가장 큰 소리부터 하나씩 빼면서 다시 들음 → 11명 찾음

**쉬운 비유**:
- COP = 한 번 사진 찍기
- T-COP = 여러 번 사진 찍어서 합성 (천체 사진처럼)
- SD-COP = 가장 밝은 별부터 하나씩 지우고 다시 사진 찍기

### 실험 1: K 스케일링 (소스 수 증가 테스트)
**"동시에 말하는 아이 수를 3명에서 20명까지 늘리면?"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| K = 3~20 | 말하는 아이 수를 점점 늘림 | 어디까지 구분할 수 있나? |
| SNR = 15 dB | 중간 정도 조용 | 실전 환경 |
| T = 256 | 보통 녹음 길이 | |
| n_trials = 10 | 10번 반복 | 평균 성능 측정 |

**핵심 결과 (3개 구간)**:
1. **K ≤ 7** (보통 구간): 모든 방법이 잘 됨. 마이크 수보다 적으니까!
2. **8 ≤ K ≤ 14** (미지수 과잉 구간): MUSIC/ESPRIT/Capon은 0%로 추락. COP는 80%+ 유지!
3. **K > 14** (초과잉 구간): COP도 힘들지만, SD-COP가 30%+ 유지

**이론**:
- MUSIC 한계 = M - 1 = 7 (고유값 분해의 수학적 한계)
- COP 한계 = ρ(M-1) = 14 (가상 배열 확장)
- SD-COP는 단계적 제거로 한계를 넘어감

### 실험 2: SNR 강건성
**"교실이 시끄러워지면 어떻게 될까?"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| K = 8 | 8명 동시 발화 | 딱 미지수 과잉 구간 (K > 7) |
| SNR = -10~20 dB | 매우 시끄러움 ~ 매우 조용 | |
| T = 128 | 짧은 녹음 | |

**SNR이 뭔가요?**
- SNR = -10 dB: 잡음이 신호보다 10배 큼 → 폭우 속에서 속삭임 듣기
- SNR = 0 dB: 신호와 잡음이 같음 → 시끄러운 식당에서 대화
- SNR = 10 dB: 신호가 잡음보다 10배 → 조용한 도서관에서 대화
- SNR = 20 dB: 신호가 잡음보다 100배 → 녹음실

**핵심**: COP는 SNR 10 dB 이상에서 이론 한계(CRLB)에 근접. MUSIC은 K=8일 때 SNR과 무관하게 실패.

**CRLB란?**: "세상에서 가장 똑똑한 컴퓨터가 있어도 이 정도 오차는 무조건 생긴다"는 이론적 최저 오차. 물리법칙에 의한 한계.

### 실험 3: 각도 분해능
**"두 사람이 얼마나 가까이 서면 구분 못 할까?"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| K = 3 | 3명 | {-Δ°, 0°, +Δ°} 위치 |
| Δ = 1°~15° | 간격을 점점 좁힘 | 15°는 쉬움, 1°는 매우 어려움 |
| SNR = 15 dB | 중간 조용 | |
| T = 512 | 넉넉한 녹음 | |

**쉬운 비유**: 눈을 감고 소리로 사람 위치를 맞추는데, 두 사람이 1m 떨어져 있으면 쉽지만, 1cm 떨어져 있으면 구분하기 거의 불가능!

**결과**: COP는 2°까지 구분, MUSIC은 5°까지만 구분. T-COP는 1°까지!

### 실험 4: 스냅샷 효율성
**"녹음을 얼마나 길게 해야 할까?"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| K = 6 | 6명 | 미지수 과잉은 아니지만 많은 편 |
| SNR = 10 dB | 살짝 시끄러움 | |
| T = 32~1024 | 매우 짧음 ~ 매우 김 | |

**T(스냅샷)란?**: 안테나가 데이터를 모으는 횟수. 사진 찍기에 비유하면:
- T = 32: 스마트폰으로 빠르게 한 장 → 흔들려서 흐릿
- T = 256: 삼각대에 놓고 촬영 → 선명
- T = 1024: 장시간 노출 → 아주 선명하지만 시간 오래 걸림

**결과**: COP는 T=128부터 안정. T-COP는 시간을 누적하니 T가 작아도 잘 됨.

**이론**: 오차 ∝ 1/√T. 녹음을 4배 길게 하면 오차가 2배 줄어듦.

### 실험 5: 큐뮬런트 차수 ρ의 효과
**"보청기 강도를 바꾸면?"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| K = 3~20 | 소스 수 변화 | |
| ρ = 1, 2, 3 | 보청기 강도 3단계 | |
| SNR = 20 dB | 조용한 환경 | ρ 효과를 깨끗이 보기 위해 |
| T = 1024 | 긴 녹음 | 큐뮬런트 추정이 정확하도록 |

**ρ 비유** (망원경):
- ρ=1 (2차): 맨눈으로 보기 → 가까운 별 7개까지 보임
- ρ=2 (4차): 쌍안경 → 14개까지 보임 (배율 2배)
- ρ=3 (6차): 천체망원경 → 21개까지 보임 (배율 3배). 하지만 설치가 오래 걸림 (데이터 많이 필요)

**핵심 결과**: K가 적을 때는 ρ=1이 제일 낫고, K가 많아지면 ρ를 올려야 함. 차수를 적응적으로 선택하는 것이 최적!

**이론**: 최대 용량 K_max = ρ(M-1). ρ를 높이면 더 많은 소스를 분리할 수 있지만, 필요한 데이터(T)가 급격히 증가.

### 실험 6: 신호 유형 의존성
**"어떤 종류의 소리가 COP로 잘 찾아질까?"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| K = 10 | 10개 소스 | 미지수 과잉 |
| 8가지 신호 유형 | 가우시안, AM, 음성, FM, 처프, QPSK, 미사일, 혼합 | |

**비유**: COP는 "특이한 소리"를 더 잘 찾습니다.
- **가우시안 (랜덤 잡음)**: 일반적인 "쉬~" 소리 → COP 48% (별로 못 찾음)
- **처프/LFM (삐~이~)**: 경보음처럼 주파수가 변하는 소리 → COP 100% (완벽!)
- **음성 (사람 목소리)**: "안녕하세요" → COP 98% (거의 완벽!)
- **FM (라디오)**: → COP 96%

**왜 이런 차이?**: COP는 "4차 통계량"을 사용합니다.
- 가우시안 신호: 4차 통계량이 0 → COP가 활용할 정보가 없음
- 비가우시안 신호 (음성, 처프 등): 4차 통계량이 크게 나옴 → COP에게 풍부한 정보 제공

**실전 의미**:
- 미사일 레이더 신호 = 처프 → COP 최적!
- 음성 마이크 어레이 = 음성 → COP 최적!
- 순수 잡음만 있으면 COP도 못 찾음 (MUSIC과 같은 수준)

### 실험 8: 다중 표적 트래킹 (생성/소멸)
**"아이들이 교실에 들어왔다가 나가는 것을 추적"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| 25 스캔 | 25번 관측 | 시간이 흐름 |
| K 변화 | 3→6→4→2 | 아이들이 들어오고 나감 |
| SNR = 15 dB | | |
| T = 256 | | |

**시나리오**:
- 스캔 1~5: 3명 있음
- 스캔 6~15: 3명이 새로 들어옴 (총 6명) → **탄생(birth)**
- 스캔 16~20: 2명 나감 (총 4명) → **소멸(death)**
- 스캔 21~25: 2명 더 나감 (총 2명)

**PHD 필터 파라미터**:

| PHD 파라미터 | 값 | 쉬운 설명 |
|-------------|-----|----------|
| `survival_prob` (p_S = 0.95) | 95% 확률로 다음에도 존재 | "이 아이가 다음 시간에도 교실에 있을 확률" |
| `detection_prob` (p_D = 0.8) | 80% 확률로 감지 | "이 아이를 찾아낼 확률". 100%가 아닌 이유: 가끔 다른 아이 뒤에 숨음 |
| `birth_weight` (w_0 = 0.15) | 새 표적 초기 가중치 | "새로 들어온 것 같은데, 일단 15% 확신" |
| `clutter_rate` (κ) | 잡음 측정 비율 | "진짜 아이가 아닌 환청의 빈도" |
| `prune_threshold` (10⁻⁵) | 너무 약한 추적 제거 | "거의 없다고 봐도 될 만큼 약한 신호는 버림" |
| `merge_threshold` (4.0) | 가까운 추적 합침 | "이 정도 가까우면 같은 아이라고 봄" |
| `extract_threshold` (0.5) | 추출 기준 | "50% 이상 확신하면 '여기 아이 있다!'고 보고" |

### 실험 9: T-COP 피드백 루프
**"추적기가 추정기를 도와주는 선순환"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| K = 6 | 6명 | {-50°, -30°, -10°, 10°, 30°, 50°} |
| 20 스캔 | 20번 반복 관측 | |
| SNR = 5 dB | 꽤 시끄러움 | 피드백 효과가 두드러지는 어려운 환경 |

**T-COP 피드백 파라미터**:

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| `alpha` (α = 0.85) | 과거 기억 유지 비율 | "이전 관측 결과를 85% 기억하고, 새 관측을 15% 반영" |
| `w_p` (prior weight) | 추적기 힌트 반영 비율 | "추적기가 '여기에 있을 거야'라고 힌트를 줌. 이것을 얼마나 믿을지" |

**피드백 루프 비유**:
1. **COP** (귀): "소리가 대충 -50° 방향에서 오는 것 같아" (부정확)
2. **PHD 트래커** (눈): "그 아이가 -50° 방향으로 걸어가고 있었으니 맞을 거야!" (힌트 제공)
3. **T-COP** (보정된 귀): "힌트를 참고하니 정확히 -50.2°야!" (더 정확해짐)
4. **PHD 트래커** (눈): "오, 더 정확한 정보를 받으니 추적도 더 잘 되네!" (추적도 개선)
5. 반복 → 점점 더 정확해짐!

**결과**: SNR=5 dB(시끄러움)에서도 피드백을 쓰면 RMSE가 빠르게 줄어듦.

### 실험 10: 움직이는 표적 추적 (교차)
**"달리는 아이들이 서로 교차할 때 헷갈리지 않기"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| 4 타겟 | 4명의 달리는 아이 | 일정 속도로 이동 |
| 30 스캔 | 30회 관측 | |
| 교차점 | ~스캔 9, ~스캔 20 | 두 아이가 같은 위치를 지남 |
| process noise = 0.3°/스캔 | 약간의 속도 흔들림 | |

**이것이 어려운 이유**:
두 아이가 같은 위치를 지나갈 때, "이 소리가 A한테서 온 건지, B한테서 온 건지" 헷갈림!

**해결 핵심 3가지**:
1. **속도 기반 병합 방지** (velocity-gated merge): 같은 위치에 있어도 속도가 다르면 다른 아이!
   - δ_m² (위치 거리 기준): 위치가 가까운지 확인
   - δ_v (속도 차이 기준): 속도도 비슷해야 같은 아이로 취급
2. **헝가리안 할당**: 예측 위치와 실제 측정의 최적 매칭 (수학적으로 가장 좋은 짝 찾기)
3. **관성 기반 예측**: "어제 오른쪽으로 달렸으니 오늘도 오른쪽에 있을 거야"

**비유**: 축구 경기에서 두 선수가 겹쳤다가 지나갈 때, 유니폼 색(=속도)을 보고 구분하는 것!

### 실험 13: 이론 검증
**"수학 공식이 정말 맞는지 실험으로 확인"**

#### 13-1: TIPP (Proposition 1) 검증
**"교차점에서 헷갈리지 않을 확률을 공식으로 예측"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| dt = 1.0 | 관측 간격 1초 | |
| σ_meas = 1° | 측정 오차 1도 | 마이크의 정밀도 |
| σ_w = 0.3° | 과정 잡음 | 아이가 약간 불규칙하게 움직임 |
| Δv = 0.1°~8°/스캔 | 속도 차이 | 두 아이의 속도 차이 |
| n_trials = 5000 | 5000번 시뮬레이션 | 정확한 통계 |

**TIPP 공식의 의미** (초등학생 버전):
```
P_TIPP = 1 - Q(Δv × Δt / (√2 × σ_p))
```

해석: **"두 아이의 속도 차이(Δv)가 크고, 측정 오차(σ_p)가 작으면 헷갈릴 확률이 낮다"**

- Δv가 크면: 한 아이는 오른쪽으로 달리고, 다른 아이는 왼쪽으로 달림 → 구분 쉬움!
- σ_p가 작면: 위치를 정확히 아니까 구분 쉬움!
- Q함수: 종모양 곡선(정규분포)의 꼬리 확률

**비유**: 두 선수가 운동장에서 교차할 때, **빨리 달리는 선수끼리** 교차하면 금방 멀어지니까 구분 쉽고, **천천히 걷는 사람끼리** 교차하면 오래 겹쳐서 구분 어려움!

**실험 결과**: 이론과 실험의 평균 차이(MAE) = 0.0166 → 공식이 거의 정확하게 맞음!

#### 13-2: 최적 Birth Weight (Proposition 2) 검증
**"COP 스펙트럼 높이로 새 표적 신뢰도를 정하면 최적인 이유"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| true_doas | -30°, -10°, 10°, 30° | 진짜 소리 4개 |
| n_trials = 100 | 100번 반복 | |
| w_0 = 0.15 | 기본 birth weight | |

**두 가지 방법 비교**:
1. **균일 birth (Uniform)**: "새로운 측정이 들어오면 무조건 같은 신뢰도(0.15)를 줌"
   - 문제: 잡음에서 온 가짜 측정도 같은 신뢰도 → 가짜 추적이 많이 생김!

2. **COP-스펙트럼 birth (우리 것)**: "COP 스펙트럼에서 높은 봉우리면 높은 신뢰도, 낮으면 낮은 신뢰도"
   - 장점: 진짜 소리는 높은 봉우리 → 높은 신뢰도 → 빨리 확인. 가짜는 낮은 봉우리 → 낮은 신뢰도 → 자동 제거

**비유**: 새 학생이 왔을 때
- 균일 방법: "이름표가 있든 없든 일단 학생으로 등록" → 유령 학생 많아짐
- COP 방법: "이름표가 선명하면 등록, 흐리면 관찰만" → 정확한 출석부!

**결과**: 균일 birth의 false rate = 1.0 (100% 잡음도 통과), COP-spectrum의 false rate ≈ 0 (잡음 거의 차단)

**이론 (네이만-피어슨)**: COP 스펙트럼 높이는 "진짜인지 가짜인지"를 구분하는 **최적의 판단 기준**임이 수학적으로 증명됨. 마치 가장 효율적인 시험 문제와 같음 — 같은 시험 시간으로 가장 정확하게 실력을 구분!

#### 13-3: 피드백 루프 수렴 (Theorem 1) 검증
**"피드백을 반복하면 반드시 정확해지는 것을 증명"**

| 파라미터 | 값 | 쉬운 설명 |
|---------|-----|----------|
| α (alpha) | 0.5~0.95 | 과거 기억 비율 |
| w_p | 0.0~0.5 | 추적기 힌트 반영도 |
| p_D = 0.8 | 감지 확률 80% | |
| n_scans = 30 | 30번 반복 | |
| SNR = 5 dB | 시끄러운 환경 | |

**수렴 공식의 의미** (초등학생 버전):
```
β = α × (1 - w_p × p_D)
```

- **β < 1이면**: 반복할수록 오차가 줄어듦 (좋음!)
- **β가 작을수록**: 오차가 빨리 줄어듦 (더 좋음!)
- **β ≥ 1이면**: 수렴하지 않음 (나쁨!)

**비유**:
- α = "어제 공부한 것을 얼마나 기억하나" (높으면 많이 기억)
- w_p = "선생님(트래커)의 도움을 얼마나 받나" (높으면 많이 도움)
- p_D = "선생님이 맞는 답을 줄 확률" (80%)

β = 0.85 × (1 - 0.2 × 0.8) = 0.85 × 0.84 = 0.714 → 매 스캔마다 오차가 71.4%로 줄어듦!
10번 반복: 0.714¹⁰ ≈ 0.035 → 처음 오차의 3.5%만 남음!

**안정성 구간 (Stability Region)**:
- α ∈ (0, 1)이고 w_p > 0이면 **항상 수렴** (수학적으로 보장!)
- 빠른 수렴: β < 0.5 → 녹색 영역 (α 작고 w_p 클 때)
- 느린 수렴: β > 0.9 → 빨간 영역 (α 크고 w_p 작을 때)

## 물리 기반 GM-PHD 트래커 핵심 파라미터 총정리

### 운동 모델 (Constant Velocity Model)
**"아이가 일정 속도로 걸어간다고 가정"**

| 파라미터 | 의미 | 비유 |
|---------|------|------|
| 상태 벡터 [θ, θ̇] | [위치, 속도] | "이 아이가 30°에 있고, 초당 2°씩 오른쪽으로 이동" |
| F (전이 행렬) | 예측 공식 | θ_next = θ + θ̇ × dt |
| Q (과정 잡음) | 속도 변화 허용 | "아이가 약간 속도를 바꿀 수 있음 (σ_w = 0.3°)" |
| H (관측 행렬) | 위치만 관측 | "소리 방향(위치)만 알 수 있고, 속도는 직접 못 봄" |
| R (관측 잡음) | 측정 불확실성 | "마이크의 정밀도 한계" |

### PHD 필터 파라미터
| 파라미터 | 값 | 비유 |

| p_D (detection) = 0.80 | 발견할 확률 | "아이를 찾을 확률 80% (가끔 놓침)" |

| w_extract = 0.5 | 추출 기준 | "50% 이상 확신하면 '여기 있다!'고 보고" |

| 단계 | 수식 | 비유 |

| 칼만 게인 | K = P⁻Hᵀ(HP⁻Hᵀ + R)⁻¹ | "예측과 관측 중 어느 것을 더 믿을지의 비율" |

| 가중치 | w = p_D·w·q / (κ + p_D·w·q) | "이 아이가 진짜 있을 확률 업데이트" |
